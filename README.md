# dev-cleanup

Move re-fetchable dev artifacts (dependencies, caches, build outputs) out of your project tree before a backup or format — without deleting anything.

Files are **moved** to an output folder preserving their relative path, so you can inspect or restore them if needed.

## Usage

```bash
./dev-cleanup.sh <source_dir> [output_dir] [--dry-run] [--compute-sizes[=all]]
```

### Options

| Argument | Description | Default |
|---|---|---|
| `source_dir` | Root folder containing your projects (required) | — |
| `output_dir` | Where artifacts are moved to | `./cleanup-output` |
| `--dry-run` | Preview what would be moved without touching anything | off |
| `--compute-sizes` | Compute and display disk space used by each matched directory. Slow on large trees. | off |
| `--compute-sizes=all` | Like `--compute-sizes` but also computes sizes for individual files | off |
| `--help`, `-h` | Show help | — |

### Examples

```bash
# 1. Quick preview — just lists what would be moved
./dev-cleanup.sh ~/projects --dry-run

# 2. Preview with sizes (slow on large trees)
./dev-cleanup.sh ~/projects --dry-run --compute-sizes

# 3. Run for real — artifacts go to ./cleanup-output
./dev-cleanup.sh ~/projects

# 4. Run with size tracking
./dev-cleanup.sh ~/projects --compute-sizes

# 5. Custom output location
./dev-cleanup.sh ~/projects ~/backup/dev-artifacts

# 6. Flags can go in any position
./dev-cleanup.sh --dry-run ~/projects ~/backup/dev-artifacts
```

### Modes and size computation

Both `--dry-run` and live mode are fast by default — neither computes disk sizes.

Add `--compute-sizes` to enable size calculation. This runs `du -sk` on every matched directory, which can **take hours on large project trees**. Use `--compute-sizes=all` to also compute sizes for individual files (`.pyc`, `.DS_Store`, etc.).

| | Default | `--compute-sizes` | `--compute-sizes=all` |
|---|---|---|---|
| **Dir sizes** | No | Yes | Yes |
| **File sizes** | No | No | Yes |
| **Total space** | Not tracked | Shown in summary | Shown in summary |
| **Speed** | Fast | Slow (runs `du`) | Slow (runs `du`) |

### Output structure

The relative path of each artifact is preserved inside the output folder:

```
~/projects/my-app/node_modules   →   cleanup-output/my-app/node_modules
~/projects/api/.venv             →   cleanup-output/api/.venv
~/projects/site/dist             →   cleanup-output/site/dist
```

## What gets moved

Everything below can be re-fetched or regenerated by running the project's install/build commands.

### Directories

| Ecosystem | Directories |
|---|---|
| **Version control** | `.git`, `.svn`, `.hg` |
| **JavaScript / Node** | `node_modules`, `bower_components`, `.npm`, `.pnpm-store`, `.next`, `.nuxt`, `.output`, `.turbo`, `.parcel-cache`, `.angular`, `.svelte-kit` |
| **Python** | `__pycache__`, `.venv`, `venv`, `.virtualenv`, `.mypy_cache`, `.pytest_cache`, `.ruff_cache`, `.tox`, `.eggs`, `htmlcov`, `*.egg-info` |
| **iOS / Xcode** | `DerivedData`, `Pods`, `xcuserdata` |
| **Flutter / Dart** | `.dart_tool`, `.fvm` |
| **Rust** | `target` |
| **Java / Android** | `.gradle` |
| **PHP** | `vendor` |
| **Ruby** | `.bundle` |
| **Build outputs** | `dist`, `build`, `out`, `.build` |
| **Caches** | `.cache`, `.sass-cache`, `.eslintcache`, `.stylelintcache` |
| **Test / coverage** | `coverage`, `.coverage`, `.nyc_output` |
| **Infrastructure** | `.terraform`, `.terragrunt-cache` |
| **Temp** | `tmp`, `.tmp` |

### Files

| Pattern | Description |
|---|---|
| `*.pyc`, `*.pyo` | Python bytecode |
| `.DS_Store` | macOS folder metadata |
| `Thumbs.db` | Windows thumbnail cache |

## What is preserved

The script **does not touch**:

- Source code
- `.env` files (environment variables / secrets)
- Documentation (`README`, docs/, etc.)
- Lock files (`package-lock.json`, `yarn.lock`, `Pipfile.lock`, `Gemfile.lock`, `composer.lock`, `Podfile.lock`, `Cargo.lock`, etc.)
- Configuration files (`.eslintrc`, `tsconfig.json`, `pyproject.toml`, etc.)
- IDE settings (`.vscode/`, project-level configs)
- Any file or directory not explicitly listed above

## How to restore

Since artifacts are moved (not deleted), you can restore anything by copying it back:

```bash
# Restore a single project's node_modules
cp -a cleanup-output/my-app/node_modules ~/projects/my-app/

# Or just reinstall from the lock file
cd ~/projects/my-app && npm install
```

## Requirements

- macOS or Linux
- Bash 4+
- Standard utilities: `find`, `du`, `mv`, `awk`
